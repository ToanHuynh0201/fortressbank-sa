Technology Decision Framework: The Hybrid Architecture

Implementing enterprise-grade SOA patterns with development velocity.

🤔
The "Why" Behind the Hybrid Model: A House Analogy

Think of SecureBank as a high-tech house. You need different kinds of doors for different purposes:

  • The Bank Vault (SOAP): For core assets like money transfers and account management, you need a vault door. It's heavy, requires multiple keys (WS-Security), has a formal, unchangeable contract (WSDL), and every interaction is logged meticulously. It's not fast or flexible, but it's incredibly secure and reliable.
  • The Front Door (REST): For daily interactions like checking your balance on a mobile app, you need a modern, convenient front door. It uses a simple, well-understood key system (OAuth2/JWT), it's flexible, and it's easy for developers to build new things that use it.

Our hybrid architecture uses the right "door" for the right job, giving us the best of both worlds: enterprise-grade security for critical operations and modern flexibility for user-facing features.

Executive Summary: The Hybrid Architecture Decision

Core Philosophy: Implement enterprise-grade SOA patterns while maintaining development velocity and system reliability.

Primary Decision: Hybrid SOAP/REST Architecture

  • Critical banking operations use SOAP for formal contracts and enterprise standards.
  • User-facing APIs use REST for modern development practices.
  • A Gateway layer unifies both approaches, providing a single entry point.

🔐 SOAP Implementation Stack

Enterprise-grade services with formal contracts, comprehensive security, and regulatory compliance.

  • Framework: Spring Web Services (Spring WS) with comprehensive WS-* standards support
  • Security: WS-Security with digital signatures, encryption, and timestamp validation
  • Transport: HTTPS with mutual TLS authentication and certificate-based identity verification
  • Contracts: WSDL-first development approach ensuring API contract stability and version management
  • Validation: Comprehensive XML Schema validation on all requests with detailed error reporting
  • Reliability: WS-ReliableMessaging for guaranteed delivery and message ordering

⚡ REST API Implementation Stack

Modern, lightweight APIs optimized for web and mobile applications with developer-friendly interfaces.

  • Framework: Spring Boot with Spring Security and comprehensive REST support
  • Authentication: OAuth2 with PKCE flow and JWT tokens for stateless authentication
  • Documentation: OpenAPI 3.0 with interactive Swagger UI for developer experience
  • Serialization: JSON with Jackson for lightweight data exchange and caching support
  • Validation: Bean Validation (JSR-303) with custom business rule validators
  • Rate Limiting: Redis-based rate limiting with per-user and per-endpoint controls

/**
 * Unified Gateway Controller - Bridges REST and SOAP architectures
 * Handles protocol translation, security, and routing
 */
@RestController
@RequestMapping("/api/v2")
@Validated
@Slf4j
public class UnifiedGatewayController {
    
    @Autowired
    private SOAPServiceRegistry soapServices;
    
    @Autowired
    private ProtocolTranslationService translationService;
    
    @Autowired
    private SecurityContextService securityService;
    
    @Autowired
    private AuditService auditService;
    
    @Autowired
    private CacheManager cacheManager;
    
    /**
     * Execute money transfer - REST endpoint calling SOAP service
     */
    @PostMapping("/transfers")
    @PreAuthorize("hasScope('transfer:execute')")
    @RateLimited(requestsPerMinute = 10)
    public ResponseEntity executeTransfer(
            @Valid @RequestBody TransferRequest restRequest,
            Authentication authentication,
            HttpServletRequest httpRequest) {
        
        // Start distributed tracing
        Span span = tracer.nextSpan().name("gateway-transfer-execution").start();
        String correlationId = UUID.randomUUID().toString();
        
        try (Tracer.SpanInScope ws = tracer.withSpanInScope(span)) {
            
            // 1. Security validation and context building
            SecurityContext securityContext = securityService.buildContext(
                authentication, httpRequest, correlationId);
            
            // 2. Input validation and sanitization
            validateTransferRequest(restRequest);
            
            // 3. Rate limiting check
            rateLimitService.checkLimit(authentication.getName(), "transfer");
            
            // 4. Convert REST request to SOAP message with full context
            TransferSOAPRequest soapRequest = translationService.convertToSOAP(
                restRequest, securityContext);
            
            // 5. Add audit trail and compliance headers
            addComplianceHeaders(soapRequest, correlationId, authentication);
            
            // 6. Execute SOAP service call with retry logic
            TransferSOAPResponse soapResponse = executeWithRetry(() -> 
                soapServices.getTransferService().processTransfer(soapRequest));
            
            // 7. Convert SOAP response back to REST format
            TransferResponse restResponse = translationService.convertToREST(
                soapResponse, restRequest.getRequestId());
            
            // 8. Update cache and analytics
            updateTransferCache(restResponse);
            analyticsService.recordSuccessfulTransfer(correlationId, restRequest);
            
            // 9. Async audit logging
            auditService.logTransferExecution(correlationId, restRequest, 
                restResponse, authentication.getName());
            
            return ResponseEntity.status(HttpStatus.CREATED)
                .header("X-Correlation-ID", correlationId)
                .header("X-Request-ID", restRequest.getRequestId())
                .body(restResponse);
            
        } catch (SOAPFaultException soapFault) {
            // Handle SOAP-specific errors
            RestErrorResponse errorResponse = translateSOAPError(soapFault, correlationId);
            auditService.logTransferFailure(correlationId, restRequest, 
                soapFault, authentication.getName());
            
            return ResponseEntity.status(errorResponse.getHttpStatus())
                .header("X-Correlation-ID", correlationId)
                .body(errorResponse);
            
        } catch (ValidationException ve) {
            // Handle validation errors
            log.warn("Transfer validation failed: {}", ve.getMessage());
            return ResponseEntity.badRequest()
                .header("X-Correlation-ID", correlationId)
                .body(RestErrorResponse.validationError(ve.getMessage()));
            
        } catch (RateLimitExceededException rle) {
            // Handle rate limiting
            log.warn("Rate limit exceeded for user: {}", authentication.getName());
            return ResponseEntity.status(HttpStatus.TOO_MANY_REQUESTS)
                .header("Retry-After", String.valueOf(rle.getRetryAfterSeconds()))
                .header("X-Correlation-ID", correlationId)
                .body(RestErrorResponse.rateLimitError());
            
        } finally {
            span.end();
        }
    }
    
    /**
     * Get account information with intelligent caching
     */
    @GetMapping("/customers/{customerId}/accounts/{accountId}")
    @PreAuthorize("hasScope('account:read')")
    @Cacheable(value = "accountInfo", key = "#customerId + ':' + #accountId", 
               unless = "#result.body.balance == null")
    public ResponseEntity getAccountInfo(
            @PathVariable @Pattern(regexp = "\\d{10}") String customerId,
            @PathVariable @Pattern(regexp = "\\d{10}") String accountId,
            @RequestParam(defaultValue = "true") boolean includeBalance,
            Authentication authentication) {
        
        String correlationId = UUID.randomUUID().toString();
        
        // Build security context
        SecurityContext securityContext = securityService.buildContext(
            authentication, null, correlationId);
        
        // Create SOAP request
        AccountInfoSOAPRequest soapRequest = AccountInfoSOAPRequest.builder()
            .customerId(customerId)
            .accountId(accountId)
            .includeBalance(includeBalance)
            .securityContext(securityContext)
            .build();
        
        try {
            // Call SOAP service
            AccountInfoSOAPResponse soapResponse = soapServices
                .getAccountService()
                .getAccountInfo(soapRequest);
            
            // Convert to REST response
            AccountInfoResponse restResponse = translationService
                .convertAccountInfoToREST(soapResponse);
            
            // Cache successful responses
            if (includeBalance) {
                cacheManager.getCache("accountBalance")
                    .put(accountId, restResponse.getBalance());
            }
            
            return ResponseEntity.ok()
                .header("X-Correlation-ID", correlationId)
                .header("Cache-Control", "private, max-age=60")
                .body(restResponse);
            
        } catch (AccountNotFoundException e) {
            return ResponseEntity.notFound()
                .header("X-Correlation-ID", correlationId)
                .build();
        }
    }
    
    /**
     * Protocol translation helper methods
     */
    private TransferSOAPRequest convertToSOAP(TransferRequest rest, SecurityContext security) {
        return TransferSOAPRequest.builder()
            .fromAccount(rest.getFromAccountId())
            .toAccount(rest.getToAccountId())
            .amount(rest.getAmount())
            .currency(rest.getCurrency())
            .description(rest.getDescription())
            .securityContext(security)
            .auditTrail(AuditTrail.builder()
                .correlationId(security.getCorrelationId())
                .originatingSystem("REST_GATEWAY")
                .requestTimestamp(Instant.now())
                .build())
            .build();
    }
    
    private TransferResponse convertToREST(TransferSOAPResponse soap) {
        return TransferResponse.builder()
            .transferId(soap.getTransferReference())
            .status(mapTransferStatus(soap.getTransferStatus()))
            .processedAt(soap.getProcessingTimestamp())
            .confirmationNumber(soap.getConfirmationNumber())
            .estimatedSettlement(soap.getSettlementDate())
            .build();
    }
    
    /**
     * Retry logic for SOAP service calls
     */
    private  T executeWithRetry(Supplier soapCall) {
        return Retry.ofDefaults("soap-service-call")
            .executeSupplier(soapCall);
    }
    
    /**
     * Error translation from SOAP faults to REST errors
     */
    private RestErrorResponse translateSOAPError(SOAPFaultException fault, String correlationId) {
        String faultCode = fault.getFault().getFaultCode();
        String faultString = fault.getFault().getFaultString();
        
        return switch (faultCode) {
            case "INSUFFICIENT_FUNDS" -> RestErrorResponse.builder()
                .errorCode("TRANSFER_INSUFFICIENT_FUNDS")
                .message("Insufficient funds for this transfer")
                .httpStatus(HttpStatus.PAYMENT_REQUIRED)
                .correlationId(correlationId)
                .build();
                
            case "INVALID_ACCOUNT" -> RestErrorResponse.builder()
                .errorCode("INVALID_ACCOUNT_NUMBER")
                .message("One or more account numbers are invalid")
                .httpStatus(HttpStatus.BAD_REQUEST)
                .correlationId(correlationId)
                .build();
                
            case "FRAUD_DETECTED" -> RestErrorResponse.builder()
                .errorCode("TRANSFER_BLOCKED_FRAUD")
                .message("Transfer blocked by fraud detection system")
                .httpStatus(HttpStatus.FORBIDDEN)
                .correlationId(correlationId)
                .build();
                
            default -> RestErrorResponse.builder()
                .errorCode("INTERNAL_SERVICE_ERROR")
                .message("An internal error occurred processing your request")
                .httpStatus(HttpStatus.INTERNAL_SERVER_ERROR)
                .correlationId(correlationId)
                .details(Map.of("soapFault", faultString))
                .build();
        };
    }
    
    /**
     * Request validation
     */
    private void validateTransferRequest(TransferRequest request) {
        if (request.getAmount().compareTo(BigDecimal.ZERO) <= 0) {
            throw new ValidationException("Transfer amount must be positive");
        }
        
        if (request.getAmount().compareTo(new BigDecimal("50000")) > 0) {
            throw new ValidationException("Transfer amount exceeds daily limit");
        }
        
        if (request.getFromAccountId().equals(request.getToAccountId())) {
            throw new ValidationException("Cannot transfer to the same account");
        }
    }
}

/**
 * Configuration for gateway routing and policies
 */
@Configuration
@EnableWebSecurity
public class GatewayConfiguration {
    
    @Bean
    public SOAPServiceRegistry soapServiceRegistry() {
        return SOAPServiceRegistry.builder()
            .accountService("https://core-banking.internal/soap/account")
            .transferService("https://core-banking.internal/soap/transfer")
            .fraudService("https://fraud-detection.internal/soap/fraud")
            .connectionTimeout(Duration.ofSeconds(30))
            .readTimeout(Duration.ofSeconds(60))
            .maxRetries(3)
            .build();
    }
    
    @Bean
    public ProtocolTranslationService translationService() {
        return new ProtocolTranslationService();
    }
}

🔍 Architecture Decision Summary

🔒 Enterprise Security (SOAP)

  • Message-level encryption and signing
  • Formal schema validation
  • Built-in reliability and ordering
⚡ Modern Integration (REST)
  • Lightweight JSON payloads

🎯 Hybrid Strategy Benefits

This hybrid architecture leverages SOAP's enterprise-grade security for critical banking operations while providing REST APIs for modern user experiences. The separation ensures regulatory compliance without sacrificing developer experience or user interface performance.
